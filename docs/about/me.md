## ✅ あなたに合った答え方（例）

### ❓「Redux より Context API の方が得意なんですね。なぜですか？」

> はい、私が関わったプロジェクトでは、主にユーザー情報やテーマ設定のような**比較的スコープが限定された状態**を共有する場面が多く、React の Context API で十分に対応できました。
>
> Context は導入コストが低く、コードもシンプルに保てるため、小〜中規模のアプリでは非常に効果的だと感じています。
>
> Redux のような複雑な状態管理やミドルウェアが必要なケースにはまだ深く触れていませんが、React の useReducer や useContext を組み合わせて、ある程度の状態管理は実現しています。

---

### ❓「Context API はどんなふうに使いましたか？」

> 具体的には、認証状態やログインユーザー情報を Context でラップし、アプリ全体で参照できるようにしました。
> `UserContext` を作って `useContext(UserContext)` でヘッダーやナビゲーションなどからユーザー名や権限情報を取り出す構成です。
>
> 状態の更新も、ログイン・ログアウト処理に合わせて `setUser` を Context から提供して、グローバルに反映できるようにしました。

以下は、React でログインユーザーの情報（認証状態・ユーザー名など）を管理する **`UserContext` の実用的なコード例**です。

### 📁 構成：

```
/contexts/UserContext.tsx
/hooks/useUser.ts
/App.tsx（or RootLayout.tsx）
```

## ✅ 1. `UserContext.tsx`（コンテキストの定義）

```tsx
// /contexts/UserContext.tsx
import React, { createContext, useState, ReactNode } from "react";

type User = {
  name: string;
  email: string;
  isLoggedIn: boolean;
};

type UserContextType = {
  user: User | null;
  login: (user: User) => void;
  logout: () => void;
};

export const UserContext = createContext<UserContextType | undefined>(
  undefined
);

export const UserProvider = ({ children }: { children: ReactNode }) => {
  const [user, setUser] = useState<User | null>(null);

  const login = (newUser: User) => setUser({ ...newUser, isLoggedIn: true });
  const logout = () => setUser(null);

  return (
    <UserContext.Provider value={{ user, login, logout }}>
      {children}
    </UserContext.Provider>
  );
};
```

---

## ✅ 2. `useUser.ts`（Context を扱うカスタムフック）

```tsx
// /hooks/useUser.ts
import { useContext } from "react";
import { UserContext } from "../contexts/UserContext";

export const useUser = () => {
  const context = useContext(UserContext);
  if (!context) {
    throw new Error("useUser must be used within a UserProvider");
  }
  return context;
};
```

---

## ✅ 3. `App.tsx`（アプリ全体に Context を適用）

```tsx
// App.tsx または RootLayout.tsx
import React from "react";
import { UserProvider } from "./contexts/UserContext";
import HomePage from "./pages/HomePage";

const App = () => {
  return (
    <UserProvider>
      <HomePage />
    </UserProvider>
  );
};

export default App;
```

---

## ✅ 4. `HomePage.tsx`（Context の利用例）

```tsx
// /pages/HomePage.tsx
import React from "react";
import { useUser } from "../hooks/useUser";

const HomePage = () => {
  const { user, login, logout } = useUser();

  return (
    <div>
      {user ? (
        <>
          <p>ようこそ、{user.name} さん！</p>
          <button onClick={logout}>ログアウト</button>
        </>
      ) : (
        <>
          <p>ログインしてください。</p>
          <button
            onClick={() =>
              login({
                name: "山田太郎",
                email: "taro@example.com",
                isLoggedIn: true,
              })
            }
          >
            ログイン
          </button>
        </>
      )}
    </div>
  );
};

export default HomePage;
```

## 💡 ポイント解説

| 機能             | 内容                                        |
| ---------------- | ------------------------------------------- |
| `UserContext`    | グローバルな認証状態を保持（ログイン情報）  |
| `useUser`        | カスタムフックで Context の呼び出しを簡略化 |
| `UserProvider`   | アプリ全体に Context を適用                 |
| `login`/`logout` | 状態変更の関数で状態の一元管理を実現        |
